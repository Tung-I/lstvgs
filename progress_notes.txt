## B. Phase 4: block-wise LapisGS LoD pyramid + per-LoD merge — COMPLETE (2026-07-01)
Goal: make each CityGS block a LapisGS layered pyramid (L0 coarse-anchored base, L1-L3
enhancement at data-factors 32->16->8->4 so L3 == the CityGS factor-4 res), then merge
per-LoD across all 9 blocks into a streamable scene-level Level-of-Detail representation,
producing a rate-distortion curve vs the flat CityGS gate.

Tooling (all in citygs/, untracked before this commit):
  - prep_lapis_block_init.py   : crop coarse PLY to a block's expanded cell -> l0_init.ply
  - scripts/run_lapis_block.sh : train one block's pyramid (lctvgs LapisGS trainer) + eval
  - scripts/run_lapis_all_blocks.sh : resumable multi-session driver, self-coordinates via
                                  atomic mkdir .train_claim locks on shared /work fs
  - merge_lapis_lod.py         : per-LoD crop-to-owned-cell + concat -> merged_lodL.ply
  - scripts/finish_lapis_lod.sh: merge + eval each LoD on the 21 val views -> RD curve
  - eval_lapis_rd.py           : per-block RD table (in-region PSNR vs owned-cell size)

## C. FINDINGS / NEGATIVE RESULT
1. Block-wise LapisGS LoD substantially UNDERPERFORMS flat CityGS. Even the top level
   LoD3, at ~2x the Gaussian count (18.7M vs 9.08M), is -5.6 dB PSNR (20.13 vs 25.78).
2. The MERGE IS FAITHFUL, not the culprit: merged LoD3 (20.13 dB) ~= mean per-block
   in-region L3 PSNR (~20.2 dB). Cropping/tiling loses almost nothing.
3. The bottleneck is PER-BLOCK LapisGS TRAINING. In-region L3 PSNR by block:
     000=21.63 001=20.89 002=21.14 003=20.36 004=21.33 005=21.28 006=17.09 007=18.62 008=21.02
   -> ~4-6 dB below flat CityGS block-finetuning across the board.
4. ROOT CAUSE = LapisGS progressive freezing. The final model = frozen coarse-anchored L0
   + frozen L1/L2 anchors + only the newly-added top-layer Gaussians are trainable. Flat
   CityGS instead jointly finetunes ALL Gaussians per block. That joint freedom is worth
   ~5 dB and is the inherent cost of the streamable-LoD (frozen-delta) constraint.
5. Secondary issues:
   - L2 -> L3 sometimes REGRESSES PSNR (000: 21.94->21.63, 006: 17.47->17.09). The full-res
     top layer trades PSNR for SSIM/LPIPS (adds high-freq detail / overfits train views).
   - block006 is pathological: all layers stuck ~17 dB, worst L0 crop -> the coarse model is
     weak in that spatial region. block006/007 are the two worst blocks.

## D. NEXT IDEAS (deferred)
